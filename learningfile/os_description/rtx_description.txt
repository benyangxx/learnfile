os_sys_init_user 用于实现 RTX 操作系统的初始化和启动任务的创建，并且使用这个函数做初始化还可以自定义任务栈的大小。（参数1：启动任务名 2：优先级 3：任务栈地址 4：任务栈大小byte）
os_tsk_create_user 用于实现 RTX 操作系统的任务创建，并且还可以自定义任务栈的大小。函数返回任务id(1:任务名 2:优先级 3：任务栈地址 4：任务栈大小)8字节对齐
os_tsk_delete_self  用于实现 RTX 操作系统的任务删除（1：删除任务的id）id=0是当前任务（空闲任务：降低系统利用率，在空闲任务实现低功耗）
os_tsk_seif 获取当前的任务ID

os_tsk_prio 用于修改任务的优先级。（1：任务id 2：修改的优先级1-254）如果任务优先级修改成功，函数返回 OS_R_OK，其余所有情况返回 OS_R_NOK，
抢占式调度：高优先级的任务立刻得到了 CPU 的控制权。如果是中断服务程序使一个高优先级的任务进入就绪态，中断完成时，被中断的任务被挂起，优先级高的那个任务开始运行
时间片调度：用于记录当前就绪的任务，并为每个任务分配一个时间片（也就是需要运行的时间长度，时间片用完了就进行任务切换）。 
合作式调度：用于当同优先级任务关闭时间片时，任务间以合作的形式运行，任务一直运行直到遇到阻塞函数切换任务。


临界段：代码的临界段也称为临界区，一旦这部分代码开始执行，则不允许任何中断打断。为确保临界段代码的执行不被中断，在进入临界段之前须关中断，而临界段代码执行完毕后，要立即开中断//全局代码段
中断锁:  中断锁就是 RTOS 提供的开关中断函数，因为 Cortex-M3/M4 的 RTX 源码中没有关闭中断的操作，所以也就没有提供开关中断函数//_disable_irq();_enable_irq();
任务锁：简单的说，为了防止当前任务的执行被其它高优先级的任务打断而提供的锁机制就是任务锁。 实现任务锁可以通过给调度器加锁或者直接关闭RTOS内核定时器（就是前面一直说的系统滴答定时器）来实现。
tsk_lock ，tsk_unlock 实现 RTX 任务的开锁和解锁：1. 函数 tsk_lock 不支持嵌套调用。2. 不允许在中断服务程序中调用 tsk_lock。

isr_sem_send用于释放信号量，调用后信号量计数值加 1。（1：OS_SEM 的变量，同时也作为 ID 标识）返回值永远是 OS_R_OK
os_sem_wait 用于获取信号量，当前信号量大于0，调用函数-1，等于0时调用函数会被挂起等待可用信号量。函数返回 OS_R_SEM 表示函数设置的超时时间范围内收到信号量可用资源 OS_R_TMO 表示超时 OS_R_OK 表示无需等待，立即获得可用信号量资源
（1：OS_SEM 的变量 2：等待时间，范围 0-0xFFFF）

互斥信号量
os_mut_init 用于互斥信号量的初始化并设置初始值（1：OS_MUT 的变量，同时也作为 ID 标识）
os_mut_wait 用于获取互斥信号量资源（1：OS_MUT 的变量 2：等待时间，范围 0-0xFFFF）
os_mut_release 用于释放互斥资源，调用此函数会将计数值减 1（1：OS_MUT 的变量，同时也作为 ID 标识。）OS_R_OK，表示互斥信号量成功释放 OS_R_NOR，表示互斥信号量的内部计数值已经是 0 或者调用此函数的任务不是互斥资源的
拥有者。
通过函数 os_mut_wait 实现互斥信号量计数值加 1，通过函数 os_mut_release 实现互斥信号量计数值减
1 操作，这样的话，这两个函数可以实现嵌套调用，但是一定要保证成对调用，要不会造成互斥资源无法
正确释放。
